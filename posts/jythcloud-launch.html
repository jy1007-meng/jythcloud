今天是一个值得记下的日子。JYTHCloud 团队正式宣布成立。

我们来自 湖南工程学院，由 吉天宇、谭海涛、胡子雄 三位在校学生共同发起。三个人的名字首字母组成了 J-Y-T-H，JYTHCloud 由此而来——Cloud 代表我们对云原生、数据驱动未来的信念。

🎯 我们的愿景：成为中小企业数据智能化转型的可靠合作伙伴，让每一家中小企业都能用上智能化的数据工具，降低技术门槛，释放数据价值。

我们是谁

三个对技术和数据充满热情的湖工学子。我们来自不同的专业背景，因为同一个想法走到了一起——用技术解决实际问题。

  • 吉天宇 — 全栈开发与系统架构，擅长将想法落地为可用的产品。
  • 谭海涛 — 数据科学与算法研究，专注机器学习模型的训练与优化。
  • 胡子雄 — DevOps 与云原生部署，保障服务的稳定与高效交付。

三个人各有所长、互补配合。从最初的技术探讨，到决定一起做点什么，再到今天正式成立团队——每一步都踩得很实。

核心技术栈

我们选择的技术方向既关注当下的工程效率，也面向未来的智能化需求：

  • Python — 团队的核心编程语言，贯穿数据分析、后端开发与算法实现。
  • PyTorch — 深度学习框架，支撑模型研发与训练。
  • Docker — 容器化部署，保证开发与生产环境的一致性。

除此之外，我们还在持续学习并实践以下技术：

# JYTHCloud 当前技术栈一览

后端: FastAPI · PostgreSQL · Redis
AI/ML: PyTorch · Hugging Face Transformers · Scikit-learn
部署: Docker · Nginx · Linux
前端: HTML/CSS/JS (vanilla) · 响应式设计
工具: Git · GitHub · VS Code · Jupyter

技术栈不是固定的，它会随着我们的项目需求持续演进。我们更看重的是解决实际问题的能力，而不是单纯地堆砌工具。

为什么是中小企业

在和身边的中小企业主、个体经营者交流的过程中，我们发现一个普遍现象：他们积累了大量业务数据（客户信息、销售记录、运营日志），但缺少有效的手段去分析、利用这些数据。大厂的 SaaS 服务要么太贵，要么功能太重，不适合小团队的实际情况。

我们想做那个「刚刚好」的解决方案——轻量、实用、价格合理，让中小企业也能享受到数据智能化带来的效率提升。

💡 我们的信念：数据不应是大公司的专利。技术应该让更多人受益，而不是制造新的壁垒。

下一步计划

团队刚刚起步，路还很长。接下来的几个月，我们重点做三件事：

  1. 完善品牌建设 — 打磨 JYTHCloud 的视觉形象和品牌定位，让更多人知道我们在做什么。
  2. 开发 SaaS 原型 — 基于我们已有的技术积累，打造第一个数据智能化 SaaS 产品的最小可行版本（MVP），并尽快让真实用户用起来。
  3. 分享学习心得 — 我们相信教是最好的学。团队会持续在主页上分享技术文章、项目复盘和学习笔记，记录成长过程，也希望能帮到同样在路上的同学。

这篇文章是 JYTHCloud 的第一篇正式公告。以后这里会陆续出现更多内容——技术教程、项目分享、踩坑记录、学习思考。敬请期待。
