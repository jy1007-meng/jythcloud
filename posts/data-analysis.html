大二上学期数据科学导论的课程作业是分析一个公开数据集。当时我想着随便找个几百行的 CSV 交差就行，结果老师说"数据集样本量不能少于 1 万条"。于是我从 Kaggle 上找了个 10 万行的电商订单数据，开始了我的第一次"大规模"数据分析。

数据分析里流传着一句话：数据清洗占整个分析过程的 80% 时间。这次算是亲身体会到了。

44| 45|

处理空值

46|
47| # 客户姓名为空的记录怎么处理？不能直接删，可能还有其他有用信息
48| # 我先看看空值比例
49| print(df["customer_name"].isna().sum())  # 3247
50| # 3% 左右，直接删除影响不大
51| df = df.dropna(subset=["customer_name"])

52| # 支付方式为空的行就用"未知"填充
53| df["payment_method"] = df["payment_method"].fillna("未知") 54|
55| 56|

处理脏数据

57|
58| # 金额列：去掉 '¥' 符号，转成浮点数
59| df["amount"] = df["amount"].str.replace("¥", "").astype(float)

60| # 日期统一格式
61| df["order_date"] = pd.to_datetime(df["order_date"], errors="coerce")

62| # 过滤负数（退款单单独处理）
63| refunds = df[df["amount"] < 0]
64| orders = df[df["amount"] >= 0] 65|
66| 67|
68| 💡 踩坑：用 pd.to_datetime() 转换日期的时候，如果遇到无法解析的格式，默认会报错导致程序中断。加个 errors="coerce" 参数之后，无法解析的日期会变成 NaT（空值），不影响其他数据的处理。这个小参数帮我省了很多事。 69|

其实这些分析本身并不难，就是 groupby、sum、mean 这些基础操作。但之前的数据清洗很关键——如果数据是脏的，再好看的分析也只是 garbage in, garbage out。

79|
80| # 月度销售额
81| monthly = orders.copy()
82| monthly["month"] = monthly["order_date"].dt.to_period("M")
83| monthly_sales = monthly.groupby("month")["amount"].sum()

84| # 画图
85| import matplotlib.pyplot as plt
86| monthly_sales.plot(kind="line", figsize=(10, 5))
87| plt.title("各月销售额趋势")

几个学到的教训

92|
    93|
  1. 先看数据长什么样再写代码。 不要上来就写复杂的分析逻辑。先 .head()、.info()、.describe() 看看数据长什么样，心里有个底。
    2. 94|
  2. 数据清洗比分析更花时间。 这次作业 80% 的时间花在了清洗上，但这也是最有收获的部分。真实的业务数据永远不会像教科书上那么干净。
    3. 95|
  3. 学会查文档比背 API 有用。 Pandas 的 API 太多了，根本记不住。重要的是知道"这个场景应该用 groupby"还是"pivot_table"，具体参数查官方文档就好。
    4. 96|
  4. 保存清洗后的中间结果。 每次清洗操作之后保存一个副本，方便回溯。不然做到后面发现前面的步骤有问题，又要重新跑一遍。
    5. 97|
98|

第一次做完完整的分析流程之后，再去看那些数据分析的课程，感觉理解深了很多。果然还是动手做项目才是最好的学习方式。
