OpenCV 安装

26|

OpenCV 是计算机视觉领域最常用的 Python 库。安装很简单：

27|
28| pip install opencv-python 29|
30|

装完之后 import 一下不报错，就算装好了。几行代码就能读取、显示、处理图片，比想象中简单不少。

31|
32| import cv2

33| # 读取图片
34| img = cv2.imread("photo.jpg")
35| print(f"图片尺寸: {img.shape}")  # (高度, 宽度, 通道数) 36|

第一个项目：人脸检测

39|

老师布置的第一个作业是用 OpenCV 检测图片中的人脸。我一开始以为要自己训练一个深度学习模型，后来发现 OpenCV 自带了一个叫 Haar 级联分类器 的东西——就是一个已经训练好的 XML 文件，直接调用就能检测人脸。

40|
41| # 加载预训练的人脸检测模型
42| face_cascade = cv2.CascadeClassifier(
43|     cv2.data.haarcascades + "haarcascade_frontalface_default.xml"
44| )

45| # 读取图片并转灰度（检测要求灰度图）
46| img = cv2.imread("group_photo.jpg")
47| gray = cv2.cvtColor(img, cv2.COLOR_BGR2GRAY)

48| # 检测人脸
49| faces = face_cascade.detectMultiScale(gray, 1.1, 5)

50| # 画框
51| for (x, y, w, h) in faces:
52|     cv2.rectangle(img, (x, y), (x+w, y+h), (0, 255, 0), 2)

53| # 保存结果
54| cv2.imwrite("result.jpg", img)
55| print(f"检测到 {len(faces)} 张人脸") 56|
57| 58|

当时我拿一张教室的合照（大概 30 个人）试了一下，检测到了 28 张脸。两个人没识别出来——一个戴了墨镜，一个侧着脸。这个准确率对于一行代码都没写的初学者来说，已经很惊喜了。

59| 60|
61| 💡 原理小理解：Haar 级联的原理其实不复杂——它通过对比图片中相邻区域的明暗变化来判断是否是人脸（比如眼睛区域比脸颊暗）。虽然是二十多年前的技术了，但简单场景下效果依然不错。 62|

进一步：尝试 OCR

65|

人脸检测做完了之后，我试了试 OCR（文字识别）。用 Tesseract 这个开源引擎，识别图片里的文字：

66|
67| import pytesseract
68| from PIL import Image

69| text = pytesseract.image_to_string(Image.open("screenshot.png"), lang="chi_sim")
70| print(text) 71|
72|

识别印刷体文字的效果还行，但对图片质量要求比较高。模糊的截图、手写字体的识别率就比较差了。不过平时用来识别截图里的文字还是挺方便的。

遇到的困难

75|
    76|
  • 图片格式搞混。 OpenCV 默认用的是 BGR 色彩空间，而 Matplotlib 用的是 RGB。用 OpenCV 读图然后用 Matplotlib 显示的时候，颜色全错了。这个问题折磨了我半个小时才找到原因——加一行 cv2.cvtColor(img, cv2.COLOR_BGR2RGB) 就好了。
    • 77|
  • 参数调优全靠试。 detectMultiScale 后面的两个参数（scaleFactor 和 minNeighbors）决定了检测的灵敏度和准确度。大了检测不到，小了误检多。没有标准答案，只能根据自己的图片反复试。
    • 78|
  • 图片路径问题。 读取图片的时候用相对路径，在不同目录下跑脚本就报错找不到文件。后来都改成了绝对路径或者 os.path.join 来拼接，省了很多麻烦。
    • 79|

对计算机视觉的印象

82|
    83|
  1. 入门门槛没有想象中高。 OpenCV 把很多复杂的算法都封装成了函数，几行代码就能跑起来。当然想深入理解原理还需要补很多数学知识，但先做出东西来再学理论，我觉得效率更高。
    2. 84|
  2. 数据质量比算法重要。 同样的代码，清晰的图片和模糊的图片识别效果天差地别。在实际应用中，先把图片拍好、拍正比调算法参数有用得多。
    3. 85|
  3. 预训练模型先用起来。 不要一开始就想自己训练模型。先用别人训练好的模型跑通流程，理解输入输出之后，再去想怎么优化。
    4. 86|
87|
