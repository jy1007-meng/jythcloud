上学期刚开始接触大模型的时候，我看了很多教程，什么 Prompt Engineering、RAG、Agent 之类的，概念一堆，但实际动手的时候还是不知道怎么下手。这篇文章记录了我从零开始调通第一个大模型 API 的完整过程，希望能帮到同样在自学的同学。

第一步：选模型

说实话，最开始我也纠结过到底用哪个。GPT-4o 很强但贵，Claude 写代码好但国内调用不方便。最后选的是 DeepSeek 的 API——原因很简单：注册送额度、文档中文、国内网络直接调。对于学生党来说，这三点就够了。

申请好 API Key 之后，我先用 curl 试了一下能不能通：

# 先测试连接
curl https://api.deepseek.com/v1/chat/completions \
  -H "Authorization: Bearer sk-xxxx" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "deepseek-chat",
    "messages": [{"role": "user", "content": "你好，请用一句话介绍自己"}]
  }'

第一次看到终端里返回那个 JSON 包着的回复时，确实挺激动的。虽然只是最简单的对话，但这是"真正的 AI"在回复我，而不是什么训练好的对话模板。

第二步：用 Python 封装一下

每次敲 curl 肯定不现实。我用 Python 写了一个简单的函数，把 API 调用包装起来：

import requests

def chat(prompt, api_key="sk-xxxx"):
    url = "https://api.deepseek.com/v1/chat/completions"
    headers = {
        "Authorization": f"Bearer {api_key}",
        "Content-Type": "application/json"
    }
    data = {
        "model": "deepseek-chat",
        "messages": [{"role": "user", "content": prompt}]
    }
    resp = requests.post(url, json=data, headers=headers)
    return resp.json()["choices"][0]["message"]["content"]

print(chat("帮我解释一下什么是 API？用小学生能听懂的话"))

跑通的那一刻确实挺有成就感的。虽然代码只有十几行，但这是我第一次真正"调用"一个大模型。

💡 踩坑记录：第一次调的时候忘了传 model 参数，报错说 model 是必填项。API 文档上写了，但我当时没仔细看。所以看文档真的很重要，别上来就复制代码跑。

第三步：加上下文，变成真正的对话

上面的代码每次只能问一个问题，模型不会记得前面说了什么。要实现真正的对话，需要把历史消息也传进去：

messages = []

def chat_with_history(user_input):
    messages.append({"role": "user", "content": user_input})
    resp = requests.post(url, json={"model": model, "messages": messages})
    reply = resp.json()["choices"][0]["message"]["content"]
    messages.append({"role": "assistant", "content": reply})
    return reply

这里有个问题值得注意：messages 越长，API 消耗的 token 就越多。在开发环境调一调没关系，但如果想做个面向用户的聊天应用，必须做上下文裁剪，比如只保留最近的几轮对话。这个我是后来踩坑才知道的。

第四步：调 Prompt 才是最难的部分

API 调通只是第一步。真正花时间的，是怎么让模型输出你想要的格式和内容。

比如我想让模型输出 JSON 格式的结构化数据：

prompt = """请从以下文本中提取信息，
以 JSON 格式返回：
{
  "人物": "xxx",
  "时间": "xxx",
  "地点": "xxx",
  "事件": "xxx"
}
文本：xxxx"""

看起来简单，但实际调试的时候你会发现：有时候它不按格式输出、有时候字段名变了、有时候多出来没让你提取的东西……只能一遍遍改提示词试。Prompt 调优本质上就是试错的过程，没什么捷径。

一些个人体会

学大模型 API 调用，最有价值的部分其实是学会"让模型按你的要求做事"。这比想象中难，也很有意思。下面几条是我自学过程中觉得比较重要的：

  1. 先调通再优化。 不要一开始就想做一个完美的产品，先让 API 返回一句话，你就已经迈出第一步了。
  2. 动手试比看教程强十倍。 看十篇 Prompt Engineering 的文章，不如自己改一次提示词跑一遍看看输出。
  3. 官方文档是第一手资料。 很多问题在文档里都有答案，比去论坛问快多了。
  4. 从简单开始。 先在命令行里跑通，再写 Web 界面，再做应用。不要一上来就想搞什么高级功能。

下一步我打算试试接入知识库，做一个能回答课程相关问题的问答机器人。等做出来了再来分享。
