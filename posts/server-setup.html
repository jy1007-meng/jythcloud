从零搭服务器：我的第一台云服务器折腾记

大二之前我对"服务器"的概念就是一台永远不会关机的电脑。直到我们团队决定做一个自己的网站，我才真正开始接触服务器运维。这篇文章记录了我从选购服务器到网站上线踩过的所有坑。

24| 25|

买服务器

26|

学生党预算有限，对比了几家云服务商，最后选了腾讯云的轻量应用服务器——2 核 4G、40G 磁盘，一年 99 块（学生优惠价）。这个配置对于学习和小型网站来说完全够用。

27|

系统选了 CentOS（当时听说比较稳），后来才知道 CentOS 停了，不过不影响使用。

28| 29|
30| 💡 选配置的建议：如果只是搭网站玩，最便宜的 2 核 2G 就够了。如果还想跑 Docker、数据库，建议 2 核 4G 起步。内存是云服务器最稀缺的资源，CPU 反而很少跑满。 31|
32| 33|

第一次登录：全是命令行的世界

34|

买完服务器之后，我用 SSH 登录了这台"云端电脑"。第一次面对 Linux 命令行的时候有点懵——平时在 Windows 上用鼠标点来点去的事情，现在全要敲命令。

35|

下面这条命令我到现在还记得很清楚：

36|
37| ssh root@你的服务器IP

38| # 登录成功之后看到的画面
39| [root@vm ~]# _ 40|
41|

第一次登录成功的时候还挺有成就感的——感觉自己真的"拥有"了一台服务器。

42| 43|

安装 Nginx

44|

要让别人能访问你的网站，需要一个 Web 服务器。Nginx 是最流行的选择，安装很简单：

45|
46| yum install -y nginx
47| systemctl start nginx
48| systemctl enable nginx

49| # 验证
50| curl http://localhost 51|
52|

看到欢迎页面的时候，我在手机浏览器里输入了服务器的公网 IP——真的能访问！虽然只是一个默认页面，但那种"我的服务器在给全世界提供服务"的感觉很难形容。

53| 54|

配置域名和 HTTPS

55|

用 IP 访问终归不方便，我买了一个域名（jythcloud.cn），然后配置了 DNS 解析指向服务器 IP。

56|

HTTPS 的配置踩了个坑：一开始我以为要花钱买证书，后来发现 Let's Encrypt 提供免费的 SSL 证书。用 acme.sh 这个工具几分钟就搞定了：

57|
58| curl https://get.acme.sh | sh
59| acme.sh --issue -d jythcloud.cn --nginx
60| acme.sh --install-cert -d jythcloud.cn --nginx 61|
62|

证书装上之后，浏览器的地址栏出现了一把小锁，网站从 HTTP 升级成了 HTTPS。这一步看似简单，但我觉得它是让你的网站"看起来正规"最关键的一步。

63| 64|

用 Docker 部署应用

65|

手动安装 Nginx、MySQL、PHP 这些环境很麻烦，而且换一台服务器就要重来一遍。后来学了 Docker，发现用容器管理这些服务方便太多了。

66|
67| # 直接一行命令跑起 WordPress
68| docker run -d --name wordpress -p 9900:80 wordpress 69|
70|

当然实际用的时候要挂载数据卷、配置网络、设置环境变量，但相比手动编译安装各种依赖，Docker 的学习成本其实低得多。

71| 72|

服务器运维日常

73|

服务器上线之后并不是就完事了，日常还需要做一些维护：

74|
    75|
  • 备份数据。 数据库和网站文件每周备份一次，防止数据丢失。
    • 76|
  • 看日志。 偶尔看看 Nginx 的错误日志，发现奇怪的就查一下。
    • 77|
  • 更新软件。 系统和 Docker 镜像定期更新一下，修安全漏洞。
    • 78|
  • 监控资源。 内存和磁盘用到了多少，心里有个数。
    • 79|
80|

听起来好像事情很多，但其实每天花不了几分钟。而且这些操作的文档和教程网上都有，遇到问题搜一下就能解决。

81| 82|

一些体会

83|
    84|
  1. 不要怕命令行。 Linux 命令行看着吓人，常用的其实就十几个命令。用多了就习惯了。
    2. 85|
  2. 善用快照和备份。 在服务器上做重要操作之前，先打个快照。搞砸了能恢复，这是最实用的建议。
    3. 86|
  3. 学生优惠很划算。 腾讯云、阿里云、华为云都有学生优惠，99 块一年，比买教程书值多了。
    4. 87|
  4. 遇到问题先搜日志。 服务器出问题了，第一时间去看日志文件。日志里通常直接告诉了你哪行代码出错了。比去论坛发帖问快得多。
    5. 88|
89|

搭完这台服务器之后，我才真正理解了"云"是什么——不是概念里的一朵云，而是真实存在的一台电脑，只不过放在别人家的机房里，通过网络远程控制它。这个理解很朴素，但够用了。
