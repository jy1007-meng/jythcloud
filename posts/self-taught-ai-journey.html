大二自学 AI 一年，我给自己的学习路线做了个复盘

一年前的这个时候，我还是个连 Pandas 的 DataFrame 都要现查文档的大二学生。对 AI 的印象停留在"ChatGPT 很牛，但怎么做的我不懂"。一年后的今天，我不敢说自己有多厉害，但至少能独立完成一个从数据处理到模型训练再到部署上线的完整流程了。

这篇复盘文章，我会按时间线梳理我自学的四个阶段，每个阶段用的资源、踩过的坑，以及最后总结的 5 条自学者建议。希望能给想自学 AI 的同学一些参考。

📌 太长不看版：打好基础（2 个月 Pandas + NumPy + Kaggle）→ 机器学习入门（3 个月吴恩达 + Scikit-learn）→ 深度学习探索（3 个月 PyTorch + Transformers）→ 工程化实战（4 个月 Flask + Docker + Nginx 部署）。如果你也在自学，可以直接跳到最后看给自学者的 5 条建议。

第一阶段：打好基础（第 1-2 个月）

Pandas NumPy Kaggle

虽然大一上过 Python 课，但课程内容基本就是输出"Hello World"、写个九九乘法表。真正处理数据的那些库——NumPy、Pandas、Matplotlib——学校根本不教。

这个阶段我没有买任何课，全是免费资源：

  • Pandas：先刷了官方 10 minutes to pandas，然后是 Pandas 中文文档 里常用的部分（read_csv、groupby、merge、apply 这些）。不求记住所有 API，够用就行。
  • NumPy：重点看了数组操作、广播机制、矩阵运算。深度学习的输入全是 Tensor，但理念和 NumPy 的 ndarray 很接近，所以这部分花的时间不多。
  • Kaggle 实战：找了一些入门级的数据集（Titanic、House Prices），试着读入数据、做 EDA、画几张图。不求建模型，只要跑通"数据→分析→可视化"这个流程就行。
💡 这个阶段的体会：不要追求"学完所有 API 再动手"。我当时的策略是——遇到不会的操作就查文档/Stack Overflow，查完记下来，下次遇到类似问题直接翻笔记。在实战中学习效率远高于通读文档。

第二阶段：机器学习入门（第 3-5 个月）

吴恩达 Scikit-learn 监督学习

从"会用 Pandas 处理数据"到"理解机器学习算法"，这中间有一道不小的门槛。很多人在这一步放弃了——因为数学太劝退。

课程选择

我选了吴恩达的 Machine Learning Specialization（Coursera）。网上评价两极分化：有人说太浅，有人说入门无敌。以我现在的眼光看，它确实偏向概念性，数学推导不够深，但作为第一门课，它非常合格。吴恩达把每个算法的直觉讲得很清楚——什么时候用、优缺点是什么、怎么调参，三个月的学习让我建立了完整的知识框架。

动手实践

学完课程后，我用 Scikit-learn 做了几个小项目巩固：

  • 线性回归：用 California Housing 数据集做房价预测，对比不同特征工程的 RMSE 差异。
  • 逻辑回归 + 决策树：做鸢尾花分类，可视化决策边界。
  • K-Means 聚类：对电商用户消费数据做 RFM 分群。
  • 随机森林 vs SVM：在一个二分类数据集上对比不同模型的效果。
⚠️ 一个常见的误区：很多人觉得"不搞懂所有数学推导就不能调库"。这是错的。我的做法是：先 70% 理解概念 + 调库跑通，遇到性能瓶颈再回去补数学。如果一上来就死磕矩阵求导和概率论，三个月后你可能还在看理论，什么项目都没做过。

第三阶段：深度学习探索（第 6-8 个月）

PyTorch Transformers NLP

机器学习基础打牢后，我开始进入深度学习领域。这个阶段对我来说是最兴奋的——因为终于能接触到"真正的 AI"了。

PyTorch 入门

为什么选 PyTorch 而不是 TensorFlow？两个原因：一是学术界和工业界的主流都在往 PyTorch 迁移；二是 HuggingFace 的 Transformers 库基于 PyTorch，学完就能直接用预训练模型。我用的是官方 PyTorch 60-minute Blitz，然后看了 d2l.ai（动手学深度学习） 的 PyTorch 版本，把前几章关于 CNN 和 RNN 的代码敲了一遍。

Transformer 与大模型

这是 2024-2025 年 AI 领域最重要的内容，没有之一。我花了很多时间理解 Transformer 架构：

  • 论文阅读：读了 "Attention Is All You Need"——第一次看只理解了大概 30%。后来又读了两遍，配合图解文章才逐渐清晰。
  • 图解视频：B 站上 3Blue1Brown 的注意力机制视频、李沐老师的 Transformer 论文精读，都看了不止一遍。
  • HuggingFace：学会了用 transformers 库加载预训练模型做推理和微调。当时第一次调通一个对话模型的感觉——真的很有成就感。
💡 Transformer 的学习建议：不用追求一次性完全理解。我的节奏是——第一遍看大致框架，知道 QKV 是什么；第二遍关注注意力计算细节；第三遍理解多头注意力、位置编码。每多看一遍，理解就深一层。

第四阶段：工程化实战（第 9-12 个月）

Flask Docker Nginx 部署

学完模型训练和推理，我发现另一个问题：模型在 Jupyter Notebook 里跑得好好的，怎么让别人也能用？这就是工程化的价值所在。这个阶段我花了 4 个月，从一个只会写 Python 脚本的人，变成了能独立部署一个 Web 服务的人。

Flask 构建 API

用 Flask 把训练好的模型包装成一个 RESTful API，写了一个简单的文本分类服务。学到了路由设计、请求校验、错误处理的套路。

# app.py - 一个简单的模型服务 from flask import Flask, request, jsonify import torch from transformers import pipeline app = Flask(__name__) classifier = pipeline("sentiment-analysis") @app.route("/predict", methods=["POST"]) def predict(): data = request.get_json() text = data.get("text", "") result = classifier(text)[0] return jsonify(result) if __name__ == "__main__": app.run(host="0.0.0.0", port=5000)

Docker 容器化

学会了写 Dockerfile 和 docker-compose，把应用和依赖打包成镜像。解决了"在我电脑上能跑啊"这个经典问题。

# Dockerfile FROM python:3.10-slim WORKDIR /app COPY requirements.txt . RUN pip install --no-cache-dir -r requirements.txt COPY . . CMD ["python", "app.py"]

Nginx 反向代理 + HTTPS

最后用 Nginx 做反向代理，配置了 SSL 证书（Let's Encrypt），把服务暴露到公网。学会了进程管理（supervisor/systemd）和日志管理。

📌 工程化的意义：很多自学者只关注模型本身，忽略了如何让别人用上你的模型。学会把模型部署成可用的服务，这个能力在找实习和工作时非常有价值。

给自学者的 5 条建议

一年下来，踩坑无数，以下是我觉得最有价值的五条经验：

1. 先跑通，再理解

这是最重要的一条。很多人学 AI 死在第一步——试图把所有数学推导搞懂再动手。我的经验是：先在 60%-70% 的理解程度下把代码跑起来，有了感性的认识，再回头看理论。理论是用来解释实践的，不是前置条件。

2. 做项目，不要只刷课

看完一个视频或一个章节，立刻动手做一个相关的小项目。哪怕只是把课程里的例子用自己的数据跑一遍，也比只看不做有效十倍。项目才是检验是否真正理解的唯一标准。

3. 不要追求完美，学会"够用就好"

你的第一个模型准确率 60% 很正常，第一个 API 可能有安全漏洞，第一个 Docker 镜像可能很大。这都没关系。先把流程跑通，再迭代优化。完美主义是自学最大的敌人。

4. 善用社区和搜索引擎

你遇到的大多数问题，网络上都已经有人解决过了。Stack Overflow、GitHub Issues、知乎、技术博客——学会精准地搜索问题是一项非常重要的能力。另外，加入一些学习社群（微信群、Discord、Reddit 的 ML 板块），在遇到困难时有人交流会好很多。

5. 保持节奏，不要跟风

AI 领域每天都有新东西出来：新模型、新框架、新论文。你不可能什么都学。我的策略是：70% 的时间巩固基础知识（Python、ML 基础、深度学习核心概念），30% 的时间了解前沿动态。基础打牢了，学新东西会很快。

写在最后

一年前，我看着各种大佬的学习路线，觉得遥不可及。一年后的今天，虽然离"大佬"还差得很远，但我至少可以说一句：自学 AI 这件事，真的没有那么难。

关键在于开始，然后保持下去。你不需要一开始就很厉害，只需要比昨天的自己进步一点点。

如果你也在自学的路上，或者打算开始，希望这篇文章能给你一点参考和鼓励。有问题欢迎在评论区交流——虽然我也不是大佬，但踩过的坑可以帮你少走一些弯路。

共勉。 一年后的你，会感谢现在开始努力的自己。
